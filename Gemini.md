4. A Simple "Battle Logic" Script

GDScript
extends Node

# Define our 2 loops
enum Types { FIRE, NATURE, WATER, LIGHT, DARK, MIND }

# The "Strong Against" Map
# Fire (0) beats Nature (1), Nature (1) beats Water (2), etc.
var strengths = {
    Types.FIRE: Types.NATURE,
    Types.NATURE: Types.WATER,
    Types.WATER: Types.FIRE,
    Types.LIGHT: Types.DARK,
    Types.DARK: Types.MIND,
    Types.MIND: Types.LIGHT
}

func calculate_damage(attacker_type, defender_type, base_damage, is_surged):
    var multiplier = 1.0
    var gives_surge = false
    
    # Check for Advantage (Double Damage)
    if strengths[attacker_type] == defender_type:
        multiplier = 2.0
        
    # Check for Disadvantage (The "Surge" Logic)
    # If the defender is actually strong against the attacker:
    elif strengths[defender_type] == attacker_type:
        multiplier = 0.75
        gives_surge = true
    
    # Apply Surge Buff if attacker was already charged
    if is_surged:
        multiplier *= 1.5
        
    return {"damage": base_damage * multiplier, "surge": gives_surge}

import random

class Monster:
    def __init__(self, name, hp, attack, defense, speed):
        self.name = name
        self.max_hp = hp
        self.hp = hp
        self.attack = attack
        self.defense = defense
        self.speed = speed
        self.atb_bar = 0

    def reset(self):
        self.hp = self.max_hp
        self.atb_bar = 0

def calculate_damage(attacker, defender):
    # Multiplier formula: Damage = Attacker_Atk * (100 / (100 + Defender_Def))
    mitigation = 100 / (100 + defender.defense)
    damage = attacker.attack * mitigation
    # Add slight variance (+/- 5%) so it's not perfectly deterministic
    variance = random.uniform(0.95, 1.05)
    return round(damage * variance)

def simulate_battle(m1, m2):
    m1.reset()
    m2.reset()
    ticks = 0
    turns_taken = 0
    
    while m1.hp > 0 and m2.hp > 0:
        ticks += 1
        # Fill ATB bars
        m1.atb_bar += m1.speed
        m2.atb_bar += m2.speed
        
        # Check for turns (simplified: only one moves per tick)
        if m1.atb_bar >= 1000:
            dmg = calculate_damage(m1, m2)
            m2.hp -= dmg
            m1.atb_bar = 0
            turns_taken += 1
            
        if m2.hp > 0 and m2.atb_bar >= 1000:
            dmg = calculate_damage(m2, m1)
            m1.hp -= dmg
            m2.atb_bar = 0
            turns_taken += 1
            
        if ticks > 5000: break # Safety cutoff

    return m1 if m1.hp > 0 else m2, turns_taken

def run_test_suite(m1, m2, iterations=100):
    m1_wins = 0
    total_turns = 0
    
    print(f"--- Simulating {iterations} battles: {m1.name} vs {m2.name} ---")
    
    for _ in range(iterations):
        winner, turns = simulate_battle(m1, m2)
        if winner.name == m1.name:

        total_turns += turns

How Devs Fight "Time Cheating"
The Gold Standard: Network Time (NTP)
Most modern mobile games don't ask the phone what time it is; they ask a server. When the app opens, it makes a quick HTTP request to an API (like Google’s or a dedicated NTP server) to get the "True Time." If the phone says it's 2028 but the server says it's 2026, the game ignores the phone.

The "Uptime" Check
Mobile OSs track how long the phone has been "on" since the last boot (System Uptime). This clock is harder to forge than the calendar clock. If the player moves the calendar forward 5 hours, but the System Uptime only progressed by 2 minutes, you’ve caught them.

The "Negative Time" Trap
If the player moves the clock forward, collects the egg, and then moves the clock back to reality, your game will suddenly see Time_Remaining = -50,000 seconds. A simple check for negative time can trigger a "Something went wrong with the chronometer!" message.

2. Creating the GlobalManager.gd (Autoload)

extends Node

# The actual function to switch scenes
func switch_scene(scene_key: String):
    if scenes.has(scene_key):
        # We use deferred to ensure the current scene finishes its logic first
        get_tree().call_deferred("change_scene_to_file", scenes[scene_key])
    else:
        print("Error: Scene key ", scene_key, " not found!")

4. Handling the "Back" Button Logic
Since your game is a mobile-style app, the "Back" button is vital. You have two ways to handle this:

The "Stack" Way (Developer Pro Tip): Keep track of the last scene in an array.

Updated GlobalManager.gd with History:

GDScript
var scene_history: Array[String] = []

func switch_scene(scene_key: String, add_to_history: bool = true):
    if add_to_history:
        # Before switching, save where we are
        var current_scene = get_tree().current_scene.scene_file_path
        scene_history.append(current_scene)
    
    get_tree().change_scene_to_file(scenes[scene_key])

func go_back():
    if scene_history.size() > 0:
        var previous_scene = scene_history.pop_back()
        get_tree().change_scene_to_file(previous_scene)
